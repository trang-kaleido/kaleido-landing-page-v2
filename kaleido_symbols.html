05 — hex-framework · beat 4 variant · with forces at vertices

The framework hexagon is a single reusable symbol. It contains all rings, spokes, vertex dots, center dot, and vertex icons. Drop it anywhere with <svg><use href="#hex-framework"/></svg>. All structural lines respond to color (currentColor). Signal elements and vertex dots are hardcoded as var(--signal) inside the symbol — they never change color.

06 — usage · how to drop a symbol into any file
Icons — on dark ground
color: var(--icon-stroke) rgba(245,241,236,0.75) — standard opacity on ink
color: var(--icon-stroke-dim) rgba(245,241,236,0.35) — dimmed / secondary on ink
Icons — on light ground
color: var(--icon-stroke-light) rgba(17,18,54,0.70) — standard opacity on paper
color: var(--icon-stroke-dim-light) rgba(17,18,54,0.35) — dimmed / secondary on paper
Nucleus icon
color: var(--signal) #5B6FA6 — always signal, on either ground
Hex mark
color: var(--paper) on dark strokes in paper, center dot in var(--signal) — hardcoded
color: var(--ink) on light strokes in ink, center dot in var(--signal) — hardcoded
Sizes
var(--icon-size-sm) = 16px inline labels, tight UI
var(--icon-size-md) = 32px card headers, section markers
var(--icon-size-lg) = 80px standalone, hero, framework display

Never create a new SVG path for a force icon in any component file. Always reference kaleido_symbols.html for the symbol definition and use <use href="#icon-[name]"/>. If a symbol needs to change, change it once here — it propagates everywhere.
